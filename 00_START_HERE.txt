🏗️ UNDERSTANDING THE APPROVAL CHAIN
================================================================================

This is the MOST IMPORTANT feature - it decides how PRs get approved.

How it works:

1. PR is submitted with grand_total = 5,50,000

2. ApprovalChain engine finds matching rules:
   - Amount between min and max?
   - PR type matches?
   - Department matches?

3. Selects chain with highest priority:
   Example: "Small Purchase (< 10L)"

4. Creates approval records from approval_sequence:
   Level 1: Assign to Manager
   Level 2: (if exists) Assign to Approver

5. Each approver must approve to move to next level

6. When all levels approved → PR marked as APPROVED

For detailed workflow, see ARCHITECTURE.md

❓ COMMON QUESTIONS
================================================================================

Q: Where do I start?
A: 1) Read QUICKSTART.md
   2) Set up locally
   3) Explore Django admin at /admin/
   4) Read ARCHITECTURE.md

Q: What is ApprovalChain?
A: Rules engine that decides who must approve a PR based on amount.
   Critical component. Defined in apps/pr/models.py

Q: How do I add a new user with approval authority?
A: 1) Admin panel → Users
   2) Create user
   3) Go to UserProfiles
   4) Set role to "Approver"
   5) Set approval_limit (e.g., 5000000 for 5L)

Q: How do users login?
A: 1) User registers/is created
   2) User navigates to /login/
   3) Enters username/password
   4) Django session created
   5) Redirected to dashboard

Q: What's the approval workflow?
A: 1) Requester creates PR with items
   2) Requester submits → Approval workflow starts
   3) ApprovalChain finds matching rules
   4) Approvers assigned based on approval_sequence
   5) Each approver approves at their level
   6) When all approved → PR marked APPROVED

Q: Can I customize approval rules?
A: YES! Go to /admin/pr/approvalchain/
   Create new chains with custom approval_sequence
   Define min_amount, max_amount
   Set requires_ipc, requires_cipc flags

Q: Where are the tests?
A: apps/[app_name]/tests.py
   Run: python manage.py test

Q: What's soft delete?
A: Records are marked deleted (is_deleted=True) but not removed
   Data preserved for compliance
   Automatically excluded from queries
   Can restore with .restore()

Q: How do I run in production?
A: See README.md "Deployment" section
   Uses gunicorn, nginx, PostgreSQL, Redis

Q: Where are the logs?
A: logs/pr_app.log (file-based rotating logs)
   Check there for errors and debugging

Q: Can I add new fields to models?
A: YES! 1) Add field to model
   2) Run: python manage.py makemigrations
   3) Run: python manage.py migrate
   4) Field is ready to use

Q: How is data secured?
A: See ARCHITECTURE.md "Security Implementation"
   - Password hashing (PBKDF2)
   - SQL injection prevention (ORM)
   - XSS prevention (template escaping)
   - CSRF protection (tokens)
   - RBAC (role-based access)
   - Audit logging (all changes tracked)
